# Epics — HubSpot Auditor

Chaque fichier d'epic est **autonome** : il contient l'hypothèse, le périmètre, les user stories et les spécifications fonctionnelles.

## Conventions de nommage

`ep[XX]-[slug-descriptif].md`

Exemples : `ep02-audit-proprietes.md`, `ep04-tableau-de-bord.md`

## Structure d'un epic file

```markdown
# [EP-XX] Nom de l'epic

## Hypothèse
Nous croyons que [construire X] pour [persona] permettra de [résultat]
parce que [hypothèse]. Nous mesurerons le succès via [métrique].

## Périmètre
### In scope
### Out of scope

## User stories
### Story 1 : [Titre]
...

## Spécifications fonctionnelles
### Règles métier
### Cas limites
### Comportements attendus

## Critères d'acceptance de l'epic
## Dépendances
## Questions ouvertes
```
